# How to Use Swarn for Game Development

## Architecture

```
<swarn_home>/              ← Swarn system (agents, logs, reports)
├── .qwen/agents/          ← Agent files
├── config/                ← Pipeline configuration
├── artifacts/             ← Reports, logs, intermediate results
└── knowledge-base/        ← Knowledge base and cases

<game_project>/            ← Your game project (generated by agents)
├── src/                   ← Source code (Game Developer, UI/UX, etc.)
├── public/                ← Assets (Asset Manager)
├── tests/                 ← Tests (QA Tester)
└── docs/                  ← Documentation (Documentation Agent)
```

## Quick Start

### 1. Configure paths

Option A — Environment variables:
```bash
# Windows
set SWARN_HOME=C:\path\to\Swarn
set GAME_PROJECT=C:\path\to\my-game

# Linux/Mac
export SWARN_HOME=/path/to/Swarn
export GAME_PROJECT=/path/to/my-game
```

Option B — Edit `config/orchestrator-config.json`:
```json
{
  "project_paths": {
    "swarn_home": "/path/to/Swarn",
    "game_project": "/path/to/my-game"
  }
}
```

Option C — Run setup script:
```powershell
.\setup.ps1 -GamePath "C:\path\to\my-game"
```

### 2. Create game project

```bash
mkdir /path/to/my-game
cd /path/to/my-game
npm init -y
```

### 3. Run pipeline

#### Option A: Full pipeline
```bash
cd <swarn_home>
qwen agent run orchestrator
```

#### Option B: Individual stages
```bash
# Concept only
qwen agent run market-analyst
qwen agent run niche-analyst
qwen agent run game-concept-designer

# Development only
qwen agent run game-developer
qwen agent run ui-ux-designer
```

#### Option C: Python script
```bash
cd <swarn_home>
python run-pipeline.py --stage 1
python run-pipeline.py --all
```

### 4. Check results

```
<swarn_home>/artifacts/
├── stage-1/
│   ├── market-analysis.json    ← Market analysis
│   ├── niche-analysis.json     ← Niche analysis
│   └── gdd.md                  ← Game Design Document
├── stage-2/
│   ├── technical-spec.md       ← Technical specification
│   └── senior-code-review.md   ← Architecture review
├── stage-3/
│   ├── code-review.md          ← Phaser code review
│   └── documentation/          ← Documentation
├── stage-4/
│   ├── test-reports.md         ← Test reports
│   ├── perf-report.md          ← Performance
│   └── localization/           ← Translations
└── final/
    ├── deployment/             ← CI/CD configs
    └── marketing/              ← Marketing materials

<game_project>/                 ← Your game (generated)
├── src/
│   ├── core/                   ← Core systems
│   ├── scenes/                 ← Scenes
│   ├── entities/               ← Game objects
│   └── systems/                ← Game systems
├── public/
│   ├── sprites/                ← Sprites and animations
│   ├── audio/                  ← Sounds and music
│   └── levels/                 ← Levels
└── tests/
    └── unit/                   ← Unit tests
```

## Workflow

### Scenario 1: New game from scratch

```bash
# 1. Run full pipeline
cd <swarn_home>
qwen agent run orchestrator

# 2. Agents work sequentially:
#    N1 (Market) → N11 (Niche) → N2 (Concept) →
#    N3 (Architect) → N15 (Senior TS) →
#    N4,N5,N6,N7,N8,N13,N17 (Parallel) →
#    N9,N10,N14,N18 (Quality) →
#    N16,N19,N20 (Release)

# 3. Result in <game_project>/
```

### Scenario 2: Improve existing game

```bash
# 1. Set path to existing project
#    in config/orchestrator-config.json

# 2. Run needed agents
qwen agent run phaser-code-reviewer
qwen agent run phaser-performance-profiler
qwen agent run qa-tester
```

### Scenario 3: Concept only

```bash
# Run Stage 1 only
qwen agent run market-analyst
qwen agent run niche-analyst
qwen agent run game-concept-designer

# Result: artifacts/stage-1/gdd.md
```

## Agent Configuration

### Where does code go?

Agents generate code in **your game project**, and reports in **Swarn**:

| Agent | Writes to | Description |
|-------|-----------|-------------|
| game-developer | <game_project>/src/ | Core code |
| ui-ux-designer | <game_project>/src/ui/ | UI components |
| level-designer | <game_project>/public/levels/ | Levels |
| audio-engineer | <game_project>/public/audio/ | Sounds |
| asset-manager | <game_project>/public/sprites/ | Assets |
| qa-tester | <game_project>/tests/ | Tests |
| documentation | <game_project>/docs/ | Documentation |

## Debugging

### View logs
```bash
# Orchestrator logs
cat <swarn_home>/logs/orchestrator.log

# Agent logs
cat <swarn_home>/logs/agents/game-developer.log
```

### View artifacts
```bash
# Stage 1 results
ls <swarn_home>/artifacts/stage-1

# Generated code
ls <game_project>/src
```

### Restart failed agent
```bash
qwen agent run game-developer --retry
```

## Tips

1. **Always check config/orchestrator-config.json** before running
2. **Git commit** in game project after each stage
3. **Use checkpoints** for recovery after failures
4. **Check artifacts/** for intermediate results
5. **Set up .gitignore** in game project for Swarn artifacts

## Troubleshooting

### Agent can't find file
- Check `project_paths` in config
- Ensure directory exists

### Circuit breaker triggered
- Check logs in `logs/agents/`
- Fix issue and restart

### Code generated in wrong place
- Check `output_artifacts` in config for the agent
- Ensure paths are absolute
